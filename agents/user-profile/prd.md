### 系统职责
这是一个回答用户信息的系统。



1. 身份画像（Identity Modeling）：建立用户基础现实世界模型。
```json
{
    "name":"", // 姓名
    "age_range":"", // 年龄范围
    "country":"", // 国家
    "city":"",   // 城市
    "native_language":"", // 母语
    "occupation":"", // 职业
    "industry":"", // 
    "education_level":"", // 文化水平
    "timezone":"" // 时区 - 可以根据国家来确定
}
```


2. 能力建模（Capability Modeling）：建立学习能力地图。

```json
{
"cefr":"A2", // cefr水平，通过听说读写分数来衡量。
"listening":62,
"speaking":41,
"reading":74,
"writing":55,
"grammar":69,
"vocabulary":58
}
```

3. 学习建模（Learning Modeling）：建立学习方式模型
```json
{
"daily_minutes":30,
"peak_hour":"21:00",
"completion_rate":0.81,
"drop_rate":0.14,
"preferred_mode":"dialogue",
"patience_score":63
}
```
4. 成长建模（Growth Modeling）：记录变化。
```json
{
"goal":"海外工作",
"current_stage":"Interview",
"completed_milestones":[]
}
```
5. 场景画像 （Scenario Profile）： 描述未来学习环境。

```json
{
"primary_track":"Global Career",
"current_scenario":"Technical Interview",
"next_scenarios":[]
}
```
6. 行为画像 Behavior Profile：预测学习行为。
```json
{
"avg_session":18,
"hint_usage":0.21,
"retry_rate":0.44,
"skip_rate":0.12,
"engagement":73
}
```

系统阶段：

#### p0：基础信息 当前阶段
- 最基础的信息采集：职业、国家、母语、当前要学习语言水平等级、听说读写水平（通俗的按照0-100分打分，不及格、及格、良好、优秀）
-【来源】注册、问卷、用户修改、评测结果

如何做？
用户输入一个个人故事背景，从故事背景中筛选出可用字段，如果没有，则系统会持续向用户提问，用户回答则填充个人资料。
个人基础资料收集完成后，需要断言用户的水平。目前可以让用户自己选择，后面会出一条类似习题或者卷子的方案让用户答题。得到英语水平。
用户的信息收集边界：最基础的数据收集后，需要主动判断目标路径中，那些数据需要收集。在此基础上持续收集。

需要出的方案有：
1. 如何交互
2. 数据架构
3. 如何展示



#### p1：学习行为采集
- 每日学习时间、最长学习时间、完成率、遗漏率、练习喜好、忍耐分数。
- 得到学习节奏、内容偏好、认知能力。



#### p2：能力收敛：记忆能力、理解能力、成长能力
- 来源： 复习结果、课程结果、课程完成率 、复习命中率、学习连续性、用户满意度

# prd更新

## 目的：用户输入内容后，根据内容的目的来收集用户的基础信息。

### 智能问卷生成引擎：

方案：
基础信息（Baseline 必填）：
```json
[
    "年龄段",
    "国籍",
    "居住地",
    "职业",
    "母语 - 根据用户使用的语言或者国籍推断",
    "身份",
    "每周可以花费在学习的时间",
    "是否有明确的目标",
    "对现状是否清晰",
    "时间截至日期",
    "其他",
]
```

拿到基础信息， 需要看用户有没有明确说明目标。
1. 先判断用户是否输入必填的基础信息，如果没有，让用户主动填写信息
2. 再判断有没有明确目标。如果没有，需要询问用户目标，需要十分明确的目标。
3. 根据目标再去反问一下现状，这时候需要AI生成问答题目，在这个目标面前，你的现在状态是什么？
> 例子1：目标是要考雅思8.5分，我就要知道你现在是什么水平，这个水平就是现状，水平也许包含，听说读写。
> 例子2：目标是理财收益达到20%每年，我要知道现在是什么水平，什么现状，你是否明白
> 例子3：目标是锻炼口才，我要知道你现在是什么水平

好的，拿到现状之后，我们就要分析现状与目标之间的距离，距离则是路径，做到这个路径需要花费的时间，心力，精力等等。
我们需要知道用户到达目标可以付出的代价，这个代价是些什么同样作为用户的信息要被询问。


1. 用户是否有明确的表达目标是什么？需要AI自行判断：不明确（需要问题确认目标）、明确（进行下一步信息收集）、



### 动态对话访谈引擎

1. 设计一个问卷状态机，定义问题的跳转规则

2. 当用户回答触发特定条件时（如满意度≤3分），自动进入追问分支

3. LLM负责根据上下文生成自然、个性化的问题表述

 设计问卷、与用户进行动态对话，并自动分析收集到的信息
