## 第二阶段： 基于第一阶段分析结果，使用LLM生成螺旋学习路径

### 第一步：语法条目评分与排序

**核心目标**：为每个EGP语法条目打分，分数越小表示越应更早进入当前等级学习路径，形成螺旋上升的学习顺序。

#### Phase 1: 基础评分（Plugin: `score`）

**系统提示词**：
```
你精通英语CEFR的设计原则以及精通语言教学，母语为中文，现在需要按照二语习得的原则为每个语法评分，最后我会根据整体数据的打分结果排序，而排序后的顺序为螺旋语法学习顺序，等价于从0到100分逐步学习该语言的学习路径。并给出打分理由。重要：打分时请使同一语法范畴（如情态、被动、时态、名词短语、从句等）的条目落在相近分数区间，以便按分数排序后形成模块连贯、减少范畴穿插的学习路径。
```

**用户提示词**：
动态构建，包含：
- 等级专属提示词
- 固定提示词
- JSON响应格式要求
- 当前语法条目信息（can_do，category，example）

**技术实现**：
- 逐条调用LLM，为每个EGP条目生成score（0-100）
- 支持重试机制（默认3次，最大12次）
- 支持限流等待和最大等待时间控制
- 输出到 `output/{lang}/<LEVEL>/latest.json` 和带时间戳的结果文件

```
命令示例：
python main.py --level A1 --lang en --plugin score
```

#### Phase 2: 全量排序（Plugin: `full-sort`）默认使用

**系统提示词**：同上

**用户提示词**：
- 一次性将全量EGP条目交给LLM做整体排序
- 强调同一语法范畴尽量落在相近分数区间
- 要求返回JSON数组，包含所有egp_id和score
- 读取 output/{lang}/{level}/prompt.json 中 其中LearningSyllabus.finalResult 作为等级专属提示词。

**技术特点**：
- 若全量返回格式非法，会先尝试让LLM修复格式
- 若仍失败，则回退到逐条评分补齐缺失项
- 自动验证返回结果（重复ID、未知ID、无效分数等）
- 输出到 `output/{lang}/<LEVEL>/full_sort_latest.json` 和带时间戳的结果文件
- 若全量排序存在明显异常，还会额外写出 `失败.json` 方便排查

```
命令示例：
python main.py --level A1 --lang en --plugin full-sort
```

### 第三步：同分处理与微调排序

**核心目标**：对第一步中分数相同的语法条目进行组内螺旋排序，确保学习路径的精准性。

#### Phase 2: 同分处理（phase2_same_score_order.py）

**运行模式**：
1. **LLM模式**：使用LLM对同分条目进行螺旋排序
2. **无LLM模式**（默认）：按EGP ID末尾数字升序排序，应该是默认模式。

**LLM提示词结构**：
```
现在进入第二步：同分组内排序。
以下语法点在第一步中拿到了相同分数`{score_value}`，请不要重新打分，而是只对这一组做组内螺旋学习排序。

排序原则：
1. 优先更基础、前置依赖更少、迁移性更强、表达频率更高的语法点
2. 若两个语法点非常接近，也必须给出稳定顺序

JSON响应格式：
{
    "ordered_ids": ["按更早到更晚排序后的 egp_id 列表"],
    "reasons": [
        {"egp_id": "GG-XXX", "reason": "中文，说明该条目为什么在这一组里更早或更晚"}
    ],
    "group_summary": "中文，概述这一组的螺旋学习排序逻辑"
}
```

**技术实现**：
- 自动识别同分条目组
- 支持断点续传（--resume参数）
- 可限制处理的组数（--limit-groups）
- 输出到 `output/{lang}/<LEVEL>/phase2_same_score_latest.json`

```
命令示例：
# 使用LLM进行同分处理
python phase2_same_score_order.py --level A1 --lang en --llm

# 无LLM模式（按EGP ID末尾数字排序）
python phase2_same_score_order.py --level A1 --lang en
```

### 第三步：学习路径抽样校验

**核心目标**：对前两步产出的学习路径进行螺旋学习逻辑验证。

#### Phase 3: 路径校验（phase3_path_check.py）

**校验原理**：
- 随机抽取多个学习路径片段（默认3次，每次20条）
- 以片段的**第一条为学习路径起点**进行评估
- LLM从螺旋递进、先简后复、模块顺序等角度评估

**校验提示词结构**：
```
你的任务：判断以下"学习路径抽样数据"是否符合上述描述。

重要约定：每段抽样数据应视为"以第一条为起点的学习路径"。即：只评估该段内从第一条到最后一条的递进是否合理、是否符合螺旋/先简后复等原则；不要考虑第一条之前是否还有其它学习内容，不要管第一条数据之前的学习路径。

JSON响应格式：
{
    "confidence": "0~1 的浮点数，表示整体上该学习路径符合上述描述的程度",
    "issues": [
        {
            "segment_label": "第几次抽样的片段，如"第1次抽取"",
            "position_or_id": "出问题的位置说明（如 rank 范围或 egp_id）",
            "description": "中文，简要说明何处不符合、为何不符合",
            "suggestion": "中文，可选的改进建议",
        }
    ],
    "overall_reason": "中文，整体说明为何给出该置信度，以及路径的优缺点概括",
}
```

**置信度阈值**：
- 默认阈值为0.85（85%）
- 若置信度低于阈值，生成警报和建议
- 输出详细的Markdown格式报告

**输出报告**：
- `output/{lang}/<LEVEL>/phase3_path_check_latest.md`
- `output/{lang}/<LEVEL>/phase3_path_check_{timestamp}.md`

```
命令示例：
python phase3_path_check.py --level A1 --lang en
```
如果置信度低于阈值，会生成告警和建议。并重新运行步骤1，直到置信度达到阈值，并且使用数据应为最新版本。
phase2_same_score_latest.json > full_sort_latest.json > latest.json

### 数据流与文件依赖关系

```
第一阶段大纲 -> 如果置信度低于阈值，会生成告警和建议。并重新运行步骤1，直到置信度达到阈值，并且使用数据应为最新版本。
    ↓ (生成)
phase1_prompt.json
    ↓ (作为输入)
main.py (--plugin full-sort)
    ↓ (生成)
latest.json 或 full_sort_latest.json
    ↓ (作为输入)
phase2_same_score_order.py
    ↓ (生成)
phase2_same_score_latest.json
    ↓ (作为输入)
phase3_path_check.py
    ↓ (生成)
phase3_path_check_latest.md (校验报告)
    ↓ (执行度低于阈值时，重新运行第一阶段并使用最新数据)

```

### 关键特性

1. **渐进式处理**：从粗到细，先整体评分再微调排序
2. **容错机制**：多级重试、格式修复、优雅降级
3. **质量保证**：多阶段校验和置信度评估
4. **可重现性**：支持随机种子、断点续传
5. **详细日志**：彩色日志输出，便于调试和监控

通过这三个阶段的处理，系统能够生成符合螺旋学习理论的高质量学习路径，并通过置信度评估确保路径的合理性。
