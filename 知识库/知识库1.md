## 一、 产品定位与核心理念

### 1.1 设计目标

Verita 从传统“背诵与测试工具”全面跃迁为 **Growth OS（成长操作系统）**。我们将语言视为个体走向未来、实现自我演化的**叙事载体（Medium）**。系统的终极壁垒不是提供一套静态的课程，而是**持续理解一个人、记录一个人，并帮助一个人在平行宇宙中“彩排未来”并最终成为那个自己的能力**。

### 1.2 核心范式转变

* **传统流程**：用户选择课程包 $\rightarrow$ 学习单词/句子意思 $\rightarrow$ 打字/选择题验证 $\rightarrow$ 错误进入错题本（原题重复）。
* **Verita 进化流程**：通过 **成长事件总线 (E1)** 捕获全量事实 $\rightarrow$ 驱动 **数字孪生 (W2)** 实时计算 $\rightarrow$ 结合 **语言知识图谱 (W1)** 与 **人生场景引擎 (W3)** $\rightarrow$ 经由 **课程/复习编译器 (L2/L7)** 动态重构、预生成并异步缓存专属课程，实现“基于掌握度与未来叙事的自适应分级进化”。

## 三、 核心引擎职责、输入输出与业务规则边界

### 3.1 第一层：事件流层 (Event Layer)

#### E1: Growth Event Stream (成长事件总线)

* **使命**：记录全量客观事实账本，只记录“发生了什么”，不持久化“当前状态”（状态交由上层实时重算）。
* **事件实例**：用户注册、完成课时、单点测试正确/错误、响应时间、连续学习天数达成、完成特定未来场景（如“技术面试剧情”）。

---

### 3.2 第二层：世界模型层 (World Model Layer)

#### W1: Language Graph (语言知识图谱) —— *第一战略壁垒*

* **使命**：重构传统线性 CEFR 体系，将零散语言知识网格化、可计算化。
* **核心节点**：
* **Token**：语言最小可学习单位（单词、短语、短句、固定搭配、语法规则、词法规则）。
* **核心要素**：规则模板、类型、CEFR 等级（A1~C2，每个级别再分简单/普通/困难）、主题、核心特征、多技能（听说读写）复用维度。


* **输入**：CEFR 官方描述、剑桥/牛津词汇与语法列表（EGP/EVP）、开源语料库及题库。
* **输出**：为 L2 课程编译器提供刚性内容依赖与知识前置/后置路径查询。

#### W2: Digital Twin (数字孪生) —— *核心资产*

* **使命**：通过 `Event Stream + 实时计算`，动态构建持续成长的人格与能力双子。
* **核心画像矩阵**：
* *基础画像*：职业、目标、偏好、国家、教育背景。
* *学习习惯*：专注时长、完成率、坚持能力。
* *四维能力模型 (基于 L6 能力引擎)*：理解能力、记忆能力、调用能力、迁移能力。


* **输入**：E1 事实事件流。
* **输出**：向 L1/L2 实时同步用户的自适应参数矩阵。

#### W3: Scenario Engine (人生场景引擎) —— *第二战略壁垒*

* **使命**：生成可能的人生路径（Possible Futures），而非预测命运。将宏观目标拆解为具象场景。
* **场景拆解树示例**：
* `目标：程序员海外出海` $\rightarrow$ `发展路线：海外工作 / 远程办公 / 创业` $\rightarrow   `核心场景集合（主题）：技术面试、签证办理、海外租房、入职团队会议`。


* **输入**：W2 提供的数字孪生目标与兴趣。
* **输出**：Scenario Tree（包含当前激活的剧情节点及上下游上下文）。

#### W4: Growth Graph (成长图谱)

* **使命**：记录用户“成为了谁”的人生图谱，提供系统最核心的情感价值与勋章成就依赖。
* **输出**：人生高光里程碑节点（如：完成了人生第一次全英文模拟面试、第一次独立用英文处理租房纠纷）。

---

### 3.3 第三层：学习引擎层 (Learning Engine Layer)

#### L1: Learning Planner (学习规划器)

* **使命**：决定“学什么”。将 W1 知识体系、W2 个人水平、W3 未来场景交叉重叠，产出未来 30 天的动态学习路线（如：Week 1 攻克海外面试，Week 2 攻克跨境跨文化沟通）。

#### L2: Curriculum Compiler (课程编译器) —— *第三战略壁垒*

* **使命**：决定“怎么学”。纯逻辑控制中心，不直接触碰大模型文本生成，只负责拼装“刚性教学大纲/编译指令单（Payload）”。
* **核心逻辑**：
1. 解析 L1 规划或 L7 复习指令。
2. 按照“**输入 $\rightarrow$ 学习 $\rightarrow$ 训练 $\rightarrow$ 输出**”的四步时序性原则，定义事件列表（Event）、活动（Activity）与任务（Task）的层级结构。
3. 锁定本次课时的 Target Tokens（目标攻克词元）、Support Tokens（支持词元），以及从 W2 获取的 **`Forbidden Tokens`（基于用户当前词汇天花板的禁用词，防知识污染）**。


* **输出**：结构化编译指令单至 L3 内容生成引擎。

#### L3: Content Engine (内容生成引擎)

* **使命**：文本与文学创作者。接收 L2 的刚性大纲约束，套上 W3 的高沉浸感故事外衣。
* **输出要求**：自动生成的长句、对话或文章，必须精准包含 Target Tokens，且绝不包含 Forbidden Tokens。

#### L4: Exercise Engine (练习生成引擎)

* **使命**：将 L3 生成的生动素材切片化、任务化。
* **核心功能**：支持将文章或对话自动转化为**统一的 Task JSON 格式**（包含：闪卡识别、单选/多选、排序练习、填空练习、口语录音、分类练习）。所有任务使用统一的内容节点格式（Audio/Video/Plain-Text/Image/Multimedia/ImageAudio）。

#### L5: Assessment Engine (评测引擎)

* **使命**：多维判断用户在交互层面的表现。
* **关键指标**：客观题正确率、口语流利度、发音精准度、写作表达质量。支持对接 Whisper、开源语音与文本大模型。

#### L6: Competency Engine (能力引擎)

* **使命**：替代二元“已掌握/未掌握”，以 0~100 分数模型精准追踪 Token 与核心要素的四维能力（理解、记忆、调用、迁移）。
* **计算模型边界**：
* *时间衰减（遗忘）*：距离上次练习越久分数自动下降，高掌握度衰减慢。
* *交互质量惩罚*：根据正确性、响应时间（超过基准线扣分）、是否使用提示进行结算。
* *增量更新机制*：只基于当前状态和本次交互计算新状态，不重算历史。
* *绝对覆写*：用户手动标记（不认识/了解/熟悉）具有最高优先级，可直接覆盖系统自动结果。
* *完美进入*：完美答对 5 次后，该单点内容才进入全局掌握表，不再纳入低级复习包。



#### L7: Review Compiler (复习编译器) —— *第四战略壁垒*

* **使命**：知识重构。摒弃传统错题原题重做的僵硬模式。
* **核心算法逻辑**：
* **基于错误模式分级诊断**：
1. *硬知识错误*（拼写、语法客观错） $\rightarrow$ 触发间隔重复、拼写题、语法填空。
2. *软语用错误*（体面性、礼貌度、文化契合错） $\rightarrow$ **触发剧情蝴蝶效应**（例如：因表达不当得罪模拟面试官 $\rightarrow$ L7 联动 W3/L2 自动生成“危机公关与补救挽回”的新场景剧本）。
3. *流利度错误*（反应慢） $\rightarrow$ 触发快速反应、句子排序、听力辨音练习。


* **基于掌握度分数动态映射复习法**：
1. *分数 <30 (陌生)*：识别阶段（直接展示原句+上下文三句话锚点复习、选择、匹配）。
2. *30-60 (学习中)*：应用阶段（展示缩略图提示、填空、句子排序）。
3. *60-90 (熟悉)*：变体复习法（生成**意义相同但用词和场景不同**的新句子）。
4. *分数 >90 (创造)*：生成复习法（给出目标短语+全新语境，让用户通过 AI 评判自主造句）。





#### L8: Learning Orchestrator (学习编排调度器)

* **使命**：高性能网关与缓存中枢。通过**影子预编译机制与异步任务队列**，在后台提前生成未来课时和复习包，将大模型生成的 15~30 秒延迟彻底消灭在后台，确保前端秒开。

## 四、 动态双闭环用户流程

### 4.1 第一闭环：日常自适应学习 (MVP 核心)

1. **命运创世（导入）**：用户进入系统，通过与 AI 导师轻量化对话，捕获职业、场景目标与体感等级（自动转化为后端 W2/W1 初始化标签）。
2. **路线编织**：L1/L2 实时联动，在首页（A3 未来仪表盘）上渲染出一条长达数月的“未来路线图”。
3. **情境摄入与训练**：拉取 L8 预生成的 Task JSON ➔ 观看带有核心词高亮的多媒体场景 ➔ 知识输入（语法规则/短语） ➔ 初步切片练习 ➔ AI 模拟实战对话。
4. **事件结算与危机埋点**：每次小任务完成触发结算事件上报 E1。若触发“软语用错误”或单点内容大幅衰减，系统在 24 小时后自动启动 L7 复习编译器。

### 4.2 第二闭环：真人/高阶 AI 破圈实战 (后续版本)

1. **剧本带入**：用户在系统内积累一定场景进度后，可预约真人外教/高阶 AI 陪练。
2. **上下文无缝传递**：系统自动将 W4 成长图谱与当前 W3 剧情简报投递给教练端（例如：“该用户正处于‘在硅谷拿不到 A 轮、团队即将解散’的推演剧情中，请你扮演刁钻投资人对其进行 15 分钟残酷面试”）。
3. **反馈反哺**：完成后由练习反馈再次生成事件，反哺 W2 数字孪生。

## 五、 客户端功能与交互规范

### 5.1 命运召唤选择页面 (Modal / 对应原 4.1 与 4.2)

* **触发方式**：全屏淡灰半透明弹窗（Modal）。
* **布局**：横向 Grid 卡片。
* **展现升级**：放弃传统的“A1 基础应用”等冷冰冰标签。卡片展示为由用户背景衍生出来的“未来命运分支卡片”（如“伦敦金融城进阶线”、“北欧定居 WLB 线”）。
* **交互效果**：hover/滑动时背景变深，文字变白，轻微缩放加阴影。选中态为深灰底加主色边框。

### 5.2 主题场景与课时节点选择页面 (对应原 4.2)

* **布局**：
* 未选中人生阶段时，全宽展示阶段列表。
* 选中后，左侧 30% 锁定为阶段列表（移动端可折叠），右侧 70% 展示主题场景卡片区。顶部带面包屑导航。


* **场景卡片控制**：匹配当前水平的正常显示；不匹配（超纲或未解锁）的卡片置灰，带锁图标，hover 时通过 tooltip 提示解锁条件。
* **课时列表**：点击卡片展开可滚动的课时列表。支持筛选（全部/未学/已学）。展示课时名、步骤数、预计耗时、进度标识。
* **联动效果**：左侧切换，右侧实时筛选并自动收起已展开的卡片。

### 5.3 课时介绍与步骤台 (对应原 4.3)

* **布局**：左侧 50% 为带渐变色的课时剧情头图；右侧 50% 为按“输入 $\rightarrow$ 输出 $\rightarrow$ 应用”严格排序的步骤卡片流（词汇 $\rightarrow$ 阅读 $\rightarrow$ 表达 $\rightarrow$ 最终任务）。
* **交互**：已完成步骤文字加删除线、图标置灰。点击卡片进入对应步骤，状态改为“学习中”。

### 5.4 学习完成与命运分叉弹窗 (对应原 4.4)

* **触发**：完成最终 Task 后自动带淡入动画和轻量音效弹出。
* **核心内容**：展示通过此场景已掌握的核心技能。
* **智能推荐**：基于主题关联和场景树分支，Grid 展示接下来的可能命运去向（课时名 + 推荐理由）。

### 5.5 A3: 未来仪表盘 (首页核心 / 对应原 4.7)

* **必备元素**：
* *置顶待办卡片*：今日核心目标（如“完成 1 个主题课时”）。
* *命运稳定系数（复习提醒）*：将传统的“错题数”包装为“未来剧情崩盘概率”。当 L6 引擎计算出核心句型掌握度跌破阈值，此卡片红灯亮起并发出危机警告（如“由于商务谈判能力衰减，你在第三年创业剧情中遭遇崩盘的概率升至 70%，请立即复习”）。
* *生存画像*：连续学习天数（火焰图标）、本周学习时长进度条。
* *继续探索入口*：大卡片，显示当前课时进度（如“会议开场 3/5”）。
* *搜索与通知*：支持模糊搜索词汇、场景名；铃铛图标聚合系统通知与社区互动。

## 六、 内容生产、数据埋点与治理规范

### 6.1 A4: CMS Studio (内容与 Prompt 工作台)

* **核心模块**：
* *知识图谱可视化编辑器*：用于录入标准化核心要素，建立依赖图谱。
* *多媒体时间轴打点器*：上传音视频，支持在特定秒数打点并关联 Token ID。
* *可视化课程与 Prompt 编排器*：树状组装任务组件，每个生成的任务必须强制绑定一个核心要素 ID 否则无法保存。
* *AI 批量出题审核流*：输入文本自动生成 Task JSON，必须经由人工审核/一键校准后方可入库。



### 6.2 埋点与结算规范 (对应原八)

* **学习交互结算事件**：每次 Task 完成立即上报。包含：`user_id`, `element_id`, `is_correct`, `response_time_ms`, `task_type`, `hint_used`。
* **掌握度变更事件**：由服务端在 L6 分数更新后触发，同步记录旧分、新分、变更原因，作为 W2 重新计算的基础。

### 6.3 第五层：治理层规范 (Governance Layer)

* **G1: Graph Governance (图谱治理)**：自动化定时任务，每日扫描并清理 W1 语言图谱中的重复节点、错误双向关系和孤立无连接节点。
* **G2: AI Safety Engine (安全治理)**：外挂刚性大模型防护壳，拦截 Prompt 注入。执行前置/后置过滤，严禁 L3 内容生成引擎输出带有政治、敏感、色情或幻觉的不合规文本，并确保其不突破 `Forbidden Tokens` 界限。
